## 1. CRITICAL BUG — `grid_backtest.py` erases realized losses on every grid rebuild

**Where:** `build_grid()`, lines 170 and 181:

```python
size = (capital * 0.95) / (price * ncell)   # uses INITIAL capital, not current equity
...
cash = capital - cost                        # RESETS cash to initial capital!
```

`cash` is set from the constant `capital` instead of the current `cash`. Every time the grid rebuilds after a stand-down or stop-loss, the account is silently refilled to $5,000.

**Reproduced:** synthetic crash 100→60 while regime lags, gate flips, bag liquidated at the bottom:

| Bar | Event | Equity |
|---|---|---|
| 150 | liquidated after crash | **$3,147.89** |
| 200 | regime returns to range → rebuild | **$4,998.38** |

A **−$1,850 realized loss vanished** at rebuild. Impact is asymmetric: the "no gate" mode builds once (unaffected), while every gated/stop-loss mode rebuilds repeatedly and gets free equity resets. **Any sweep run with this code overstates exactly the modes you're trying to validate — all Phase-0 verdicts to date are invalid and must be rerun after the fix.**

**Fix:**
```python
def build_grid(price, i):
    nonlocal ...
    equity_now = cash + inv * price          # carry real equity through
    band = ...
    size = (equity_now * 0.95) / (price * ncell)
    ...
    cash -= cost                             # spend from actual cash, don't reset
    ...
```
(Also cap `holding` pre-buys by available cash.)

## 2. `grid_backtest.py` — same-bar regime lookahead flatters the gate

`internal_regime()` classifies bar *i* using bar *i*'s **close** (ER window ends at `cl[i]`; flash = that bar's own move), and `simulate()` then applies `regime[i]` to fills **inside bar i**. In reality you only know a bar is a flash/trend bar after it closes — so the sim avoids buying into crash bars it couldn't have seen coming. The docstring claims the regime is "lagged, causal"; only the analyst-CSV path is (`regime_at` uses strictly-before timestamps). The `persist=24` hysteresis blunts this for trend, but **flash flips immediately** — the most lookahead-sensitive case.

**Fix:** gate on `regime[i-1]` (shift the internal regime by one bar). Cheap and honest.

Minor, same file: `regime_at()` rebuilds the key list per call (O(n²) — precompute once); ATR warm-up uses first-n mean (trivial forward leak, ignore); intrabar fill model is fine (a cell can't buy and sell in the same bar — verified).

## 3. Intraday-TSM live/trial — cost model undercounts by ~half

- `intraday_tsm_forward.py` charges `FEE_BPS = 10` **per round-trip** (`net = gross - fee`, once). The live executor enters and exits with **market orders** — two taker fills = **~20 bps** (15 with BNB). The trial's scorecard flatters every trade by ~5–10 bps. Given the whole postmortem lesson is "the toll kills thin edges," the forward test should charge the real toll: set `FEE_BPS = 20` (or your actual tier ×2) or the "live-confirmed" bar is soft.
- **Entry-price mismatch:** the trial commits at the 07:00-close price; the live bot fills at market at ≥08:05. That 5+ minute drift + spread is untracked edge slippage. Either enter with a limit near the committed price, or log the gap per trade (`compare_live_vs_trial.py` should assert on it daily, not on demand — see §5).
- **SHORTs on spot:** `exchange()` builds ccxt binance with no `defaultType` → **spot**. A spot "sell" only works if you hold the base asset; it is not a short. Either your SHORT entries fail quietly ("ORDER FAILED, skipped" — silently biasing the live book long-only) or they're selling pre-held testnet balances, which isn't short economics (no borrow cost, and P&L math assumes a true short). Use USDT-M futures testnet for shorts, or log SHORT days as shadow-only and say so on the dashboard.
- **Live P&L excludes fees** (`pnl = sign*(fill-entry)*amount`) — same undercount, on the headline number.
- **Local `tsm_forward.db` is stale** (8 FLAT rows, one day, 2026-06-17, pre-realignment basket). The record of truth lives only on the server. Schedule `pull_server_data.ps1` so a server loss can't take the track record with it.

## 5. What could have been done better (postmortem review)

The postmortem discipline is genuinely strong — IS/OOS, placebo, breadth, deflated Sharpe, pre-commitment. The failures below are about *sequencing and shared infrastructure*, not rigor:

1. **The 5-year test came last instead of first.** Intraday-TSM and stablecoin-flow each burned weeks on a 1-year result that a 5-year run (same code, one flag) would have killed on day one. Rule: **no strategy gets a second session of work until it has survived the full multi-regime history.** The cheap test goes first.
2. **The cost wall was discovered empirically five times.** A 20-line "breakeven calculator" — given fee, horizon, and realistic hit-rate, what per-trade edge do I need? — would have closed the entire fast-scalp family before Kronos was built. Make it the first cell of every new strategy notebook.
3. **Every backtest re-implements its own metrics.** ~20 scripts each have their own equity/Sharpe/fee logic — that's how the §1 bug slipped in, and how inconsistent fee assumptions (§4) crept across scripts. Extract one shared, unit-tested `validation.py` (equity accounting, IS/OOS split, placebo, breadth, per-regime split, worst-month) and make every strategy import it. One audited accounting engine instead of twenty.
4. **The live/trial wiring bug was caught by eyeballing a divergence.** `compare_live_vs_trial.py` exists but runs on demand. Make reconciliation a **daily cron that alerts loudly** when live ≠ trial (direction mismatch or fill-gap > X bps). The "one brain" rewire was the right fix; auto-reconciliation is the guard that keeps it fixed.
5. **Repeated re-mining of buried signals.** Runs A–D kept resurfacing `conviction`-style rules a dedicated test had already killed. Keep a small **graveyard registry** (signal, date killed, killing test); the search harness refuses to re-propose registered corpses. Turns discipline from memory into code.
6. **Backtests aren't pinned to commits.** Results in the tracker can't be reproduced exactly because the code kept moving (grid sweep results would currently be from uncommitted code). Log `git rev-parse HEAD` + full config into every results file.

## 6. Creative directions (each subject to the same honesty bar)

**Attack the cost side, not the signal side.** The postmortem treats 15–20 bps as a law of nature; it's a parameter. Edges you already proved *exist but don't clear the toll* (OFI, TSM at 10 bps) become viable if the toll drops: maker-only execution for TSM entries/exits (limit at the committed price — you're already committing pre-price, so resting a limit is natural), venues/pairs with zero-fee or maker-rebate promos, and fee-tier/BNB optimization. Cheapest alpha available: you already found the signals, you just can't afford them yet.

**Grid: hedge the bag instead of dumping it ("freeze + hedge").** Your two gate modes are dump (realizes the loss, taker fees) or freeze (bag rides the trend down). Third mode: on trend_down, keep the spot grid frozen and **short an equal notional perp** — inventory goes delta-neutral, the trend can't hurt it, and you often *collect* funding while hedged (shorts get paid in panics). Unwind the hedge when range returns. Turns the grid's fatal flaw into a funding-carry position. Backtest it as a fourth gate mode.

**Covered calls on grid inventory — the VRP harvest you already earned.** Your two strongest results: VRP is real (+9.7 vol-pts, 74% of days) but naked short-vol gets killed by tails; and a grid structurally accumulates inventory. Combine them: **sell calls against inventory the grid already holds.** Covered calls have no left-tail beyond the coins you own — the "fat-tailed endpoint" that killed the condor caps your *upside*, not your survival. The grid earns rungs in chop; the calls monetize the vol premium on the bag while it waits. Deribit read-only key is already working. This is the defined-risk VRP harvest the postmortem said didn't exist.

**Test the fade you already discovered.** The careful-bets ladder proved the *biggest* morning moves mean-revert in the afternoon (OOS Sharpe went 0.11 → −0.42 as you selected for them). That monotone *inversion* is itself a candidate signal: **fade the top-decile morning move** instead of riding it. You've effectively run half the experiment; run the mirror before anyone else re-mines it. Pre-register the config, 5-year bar, placebo, breadth — if it's noise, it dies in a day.

**Regime classifier as allocator, not switch.** The analyst's one validated skill is range/trend/flash discrimination. Instead of gating one strategy on/off, use it to **allocate one pot across three sleeves**: range → grid, trend → buy-and-hold (or TSM), flash → cash/hedge. Buy-and-hold made +40% over the window your timing strategies lost money in — a classifier that merely keeps you *in the market during trends* and *in the grid during chop* has a much lower bar to add value than one that must beat the market alone. Backtest = same 5-year bar, benchmark = 100% buy-and-hold.

**Funding-carry as the grid's cash yield.** Idle quote-side capital (the grid holds ~half in USDT waiting for buys) can sit in the delta-neutral funding-carry you researched (~19%/yr headline, realistically less) instead of earning zero. Even a few percent on idle balances materially lifts a strategy fighting for bps.

## 7. FIXES APPLIED (2026-07-03, same day — all regression-tested on synthetic data)

**`grid_backtest.py`**
- `build_grid()` now carries real equity through rebuilds (`equity_now = cash + inv*price`), sizes from current equity, and pre-buys only the inventory *shortfall*, capped by cash. Regression: the −$1,850 crash loss now survives the rebuild (residual −$1.02 = re-setup fees).
- Internal regime is lagged one bar (`lag1()`) — bar *i* is gated on what was knowable at *i−1*'s close. Analyst-CSV path unchanged (already causal). `regime_series()` replaces the O(n²) lookup.
- **New `hedge` gate mode** (4th mode, in single-run and sweep): on stand-down, pause the book and short a perp for the inventory notional — delta-neutral instead of dumping (exit's flaw: sells the bottom) or riding the bag down (freeze's flaw). Re-centers DGT-style if price left the band when range returns. Modeled conservatively: taker on both hedge legs, no funding income credited. Regression: equity flat through a 40% crash while hedged; in a V-bounce freeze > hedge > exit, as theory predicts — no free money anywhere.

**`intraday_tsm_forward.py`** — `FEE_BPS` 10 → 20 (two taker legs, the toll the live book actually pays). Old settled rows keep their gross; nets recomputable.

**`intraday_tsm_live.py`** — realized P&L now subtracts both-leg taker fees (`TSM_TAKER_BPS`, default 10); `EXCHANGE_MARKET_TYPE` env sets ccxt `defaultType` (set `future` on USDT-M testnet for real shorts); loud warning when a SHORT is placed in spot mode. **Deploy note: these fixes only take effect on the server once you pull them there.**

**`fade_backtest.py` (new)** — the pre-registered morning-fade mirror test: fade top-decile morning moves (trailing 60d q0.90, strictly prior days), split-close → day-close, 20 bps, 7 coins, IS/OOS + breadth + worst month, with q0.80/q0.95 and the ride-mirror as context rows. Harness validated three ways on synthetic data: reverting world → PASS 7/7 (ride-mirror symmetric loser), random and momentum worlds → FAIL. The verdict rule is in the docstring: if q0.90 fails, graveyard it — no shopping in the neighbors.

**To run on your machine** (sandbox can't reach Binance; ~5yr hourly fetches take a while):


## 8. DASHBOARD APP AUDIT (added later on 2026-07-03)

Scope: `dashboard_server.py`, `dashboard.html`, deploy units. Architecture is sound (read-only journal, sandbox-guarded close-only trading, background candle thread, deploy docs correctly keep port 8765 off the internet via UFW + Tailscale).

**Found and FIXED same day:**

1. **Manual-close P&L ignored fees** while the bot's auto-exit (post-fix) subtracts them — the dashboard's "you vs bot" scoreboard was structurally biased toward manual closes. Now subtracts both-leg taker fees (`TSM_TAKER_BPS`), matching the auto path.
2. **`/close` was CSRF-able.** A form POST is a "simple request": any website open in a browser on a machine that can reach the dashboard (LAN/Tailscale mesh) could silently POST `symbol=__ALL__` and flatten every position. Now requires the `X-Kronos-Close: 1` header (forces a CORS preflight that is never approved; the dashboard's own fetch sends it; curl users add `-H "X-Kronos-Close: 1"`).
3. **`/run-search` and `/run-geometry` had no concurrency guard** — refresh-spamming stacked parallel 5–10 min subprocesses. Now single-flight behind a lock; concurrent callers get a "already running" error.
4. **Journal-path double close.** The manual close places a market order but can't update `journal.db` (bot-owned, opened read-only), so the position stays OPEN in the UI and a second click sent a second market order (on spot that can dump unrelated balance). Now an in-session guard refuses a re-close and explains why.
5. `Tuple` was missing from the typing import (masked by `from __future__ import annotations`). Added.

**Noted, not changed:**

- **No auth by design** — acceptable *only* while UFW + Tailscale hold; worth re-running `ufw status` on the server after any re-provision. A one-line bearer token would let it survive a config mistake.
- Log/DB strings render via `innerHTML` unescaped (~57 sites). Sources are your own bot log and ccxt error strings, so risk is low on a private mesh — an `esc()` helper would close it fully.
- Marks and manual-close fallback prices come from **testnet klines** — consistent with testnet trading, but don't read the unrealized P&L as mainnet-accurate.
- `_trade_exchange()` caches a failed init until restart (fix `.env`, then restart the service).
- **Deploy note:** all fixes take effect on the server only after you pull + `systemctl restart kronos-dashboard`.

## 10. ANALYST × INTRADAY INTEGRATION (added 2026-07-04) — two decision tools

Prompted by two observations: "the market pumped and the bot gated everything out" and "the strategy is locked to one hour." Both get evidence, not live tweaks:

1. **`tsm_split_study.py`** — generalizes TSM to a rolling session (prior 8h signal → next 16h hold) and tests entry hours 0/4/8/12/16/20 UTC with the live gates and real 20 bps, 5yr, IS/OOS + breadth. Pre-registered read: broad green across hours → a multi-hour ensemble is justified (via shadow forward test); only 08:00 green → the live hour was likely luck (evidence, not license to hop); nothing green → matches the failed 5yr backtest. Validated on synthetic momentum (6/6 green) and noise (0 green) worlds.
2. **`regime_allocator_backtest.py`** — the honest pump-catcher. Missing pumps is *by design* for TSM (it trades session momentum, not trends); the fix is a separate sleeve driven by the regime classifier (the analyst's one validated skill): `trend-rider` (long only in trend_up) and `default-long` (long unless trend_down/flash) vs buy & hold, net of flip costs, 5yr IS/OOS, per-coin. Pre-registered bar: beat B&H OOS risk-adjusted on most of the basket or it doesn't ship. Supports `--regime-csv` for the analyst's own regime export.

**Explicitly rejected:** loosening TSM's gates or moving its hour because of a felt missed pump — that's the postmortem's mining trap. If the split study says the edge is broad, the upgrade path is a multi-hour *shadow forward test first*, same as everything else.

**Pump confirmation from the analyst (added 2026-07-04).** The analyst already ships one: its regime signal is its only RELIABLE direction call (SIGNAL_CARD: BTC 1h/24-bar 66% vs 52% base, lift 1.25, stable both halves; ETH 63%), `trend_up`/`breakout` = confirmed pump, with a confidence score, computable any day — no fixed hour. The allocator is now wired to it: `load_analyst_regime()` reads `data_cache/analyst_lean.csv` (causal export, 2021-12→now, all 7 coins; 4h version for BTC/ETH), applies the **+1 bar-period shift the export's own docstring requires** (row for day D is computed on D's close — using it during D would be lookahead), counts `breakout` as confirmation, and `--min-conf N` downgrades low-confidence calls to unconfirmed. Verified on synthetic CSV: label unusable during its own day, usable the next, low-confidence downgraded.

```
.venv\Scripts\python.exe regime_allocator_backtest.py --days 1650 --regime-csv data_cache\analyst_lean.csv
.venv\Scripts\python.exe regime_allocator_backtest.py --days 1650 --regime-csv data_cache\analyst_lean.csv --min-conf 0.6
```

Primary (pre-registered) = raw labels, `--min-conf 0`; the 0.5/0.6/0.7 runs are robustness context only. If a sleeve passes the bar, the live path is polling the analyst API (`GET /regime/{symbol}` returns label + confidence + reasons) — via a **shadow forward test first**, never straight to orders.
